Storage Manager for Proxmox (SMP) 为使用 NetApp ONTAP 存储系统的 Proxmox 环境中的虚拟机提供全面的存储配置和生命周期管理。SMP 使管理员能够直接管理存储，通过自动配置、备份、还原和克隆功能简化操作。



== 了解 Storage Manager for Proxmox

Storage Manager for Proxmox (SMP) 为使用 NetApp 存储系统的 Proxmox 环境中的虚拟机提供端到端的存储管理。通过允许管理员直接管理存储，SMP 简化了 Proxmox 环境中的存储和数据管理任务。



=== 主要特点

* *端到端存储管理*：SMP 为虚拟机提供完整的生命周期管理，从调配存储到备份和恢复。
* *直接存储管理*：Proxmox 管理员可以直接管理存储，简化操作并降低复杂性。
* *独立安装程序*：SMP 作为独立安装程序交付，通过 API 无缝连接到 Proxmox 节点。




=== 功能

SMP 的每个组件都旨在提高存储管理效率：

* *存储配置*：轻松为 Proxmox 主机和相关虚拟机分配和管理存储资源。
+
** *NFS 存储配置*：设置和配置要与 Proxmox 环境一起使用的 ONTAP NFS 存储。
** *iSCSI 存储配置*：实施具有多路径的厚 LVM，以实现强大而高效的 iSCSI 存储管理。


* *备份和恢复*：执行可靠的备份和快速恢复，以确保数据的完整性和可用性。
+
** *崩溃一致性和文件一致性备份*：通过设置崩溃一致性和文件一致性备份机制来确保数据完整性。
** *VM 还原操作*：执行虚拟机还原流程以恢复数据并维护系统可用性。


* *存储效率*：ONTAP 提供全面的存储效率技术套件，旨在最大限度地减少数据所需的存储。这些技术不仅可以降低存储消耗，还可以提高系统性能，从而提供：
+
** *快速 VM 配置*：简化虚拟机配置，以加快部署。
** *更快的数据集拷贝*：加快数据集的复制，实现高效的数据管理。






=== 前提条件

在继续之前，请确保您已完成以下操作：

* 工作的 Proxmox Virtual Environment (PVE) 9.x 或更高版本集群。
* ONTAP All Flash FAS (AFF) 存储，在 SVM 级别启用 NFS 和 iSCSI 协议。有关说明，请参见 link:proxmox-ontap-nfs.html["使用 ONTAP 配置 NFS 数据存储"] 和 link:proxmox-ontap-iscsi.html["使用 ONTAP 存储为 iSCSI 配置 Proxmox VE 9.x"]。
+
[NOTE]
====
对于 iSCSI LUN 配置，应启用以下内容：

*在所有 Proxmox 主机上安装多路径和 iSCSI：*

[source, shell]
----
apt update
apt install open-iscsi multipath-tools -y
systemctl enable --now iscsid multipathd
----
*配置 iSCSI 启动程序并通过 iSCSI 连接（在 Proxmox 主机上）：*

创建或更新 `/etc/multipath.conf`：

[source, shell]
----
touch /etc/multipath.conf
----
或使用：

[source, shell]
----
cat << 'EOF' > /etc/multipath.conf
defaults {
   user_friendly_names yes
   find_multipaths yes
}
blacklist {
   devnode "^sd[a]$"
}
EOF
----
====
* Proxmox 主机和 ONTAP 存储可从 SMP 主机进行网络访问。

=== 卷配置选项

当 SMP 配置新的 ONTAP FlexVol 卷和 LUN 时，会自动应用几个默认配置值：

[cols="2*"]
|===
| ONTAP 设置 | 默认值 


| Snapshot 预留 (-percent-snapshot-space) | 0% 


| 分数预留 (-fractional-reserve) | 0% 


| 访问时间更新 (-atime-update) | 已禁用 


| 最小预读数 (-min-readahead) | 已禁用 


| 计划快照 | 无 


| 存储效率 | 已启用 


| 卷保证 | 无（精简配置） 


| 卷自动调整 | grow_shrink 


| LUN 空间预留 | 已禁用 


| LUN 空间分配 | 已启用 
|===


=== NConnect 行为

将 NFS 存储与 Proxmox VE 一起使用时，请注意 Linux NFS 客户端（由 Proxmox VE 的基本操作系统 Debian 使用）中的特定行为。Linux 内核可能会重用首次在节点上挂载 NFS 导出时的挂载参数。如果使用不同的挂载选项再次挂载相同的导出，则可能不会应用这些新选项。

作为最佳做法，创建和装载单独的导出并使用不同的 NFS 服务器 IP 地址，例如不同的数据 LIF。SMP 自动检测现有连接及其关联的 nconnect 值，并将相应的 nconnect 值应用于任何新连接。



== 保护虚拟机

当虚拟机在 ONTAP 支持的 NFS 数据存储上运行时，最简化的保护方法是让 Proxmox 建立文件系统一致性，而 ONTAP 在存储层捕获实际备份点。ONTAP 快照提供数据存储卷的快速、节省空间的时间点副本，以后可用于恢复整个数据存储、单个文件或 VM 内容。

SMP 为崩溃一致和 VM 一致的数据保护和克隆管理提供了一个简单的协调器。此软件通过创建可应用于多个作业的备份策略来简化备份、还原和克隆生命周期。这些策略可以定义计划、保留、复制和其他功能。它们允许可选地选择文件一致性快照；这种方法使用 QEMU 在拍摄 ONTAP 快照之前停止 I/O 的能力。

SMP 允许保护整个数据存储库，其中一个 ONTAP NFS 卷包含多个虚拟机、可能跨越多个数据存储库的虚拟机组或单个虚拟机级别。

** QEMU 来宾代理冻结/解冻，用于来宾文件系统静止和 Windows VSS 集成。

=== 还原工作流详细信息

还原工作流程因基础存储类型而异，涉及以下关键步骤：

. *定义还原范围*：选择保护目标是数据存储区（ONTAP 卷）、资源组（多个卷或 VM）还是单个 VM。
. *执行还原操作*：VM 还原过程因存储协议而异：
+
** *基于 NFS 的恢复*：在 NFS 上恢复卷或数据存储时，SMP 会在目标卷上启动卷快照恢复操作。对于托管在 NFS 卷上的虚拟机，SMP 会验证还原参数，包括 VM ID、快照名称、站点 ID 和存储类型。如果 VM 正在运行，SMP 通过轮询其状态并发出停止命令来停止它。然后，系统使用文件级还原操作从快照中还原 VM 磁盘。
** *基于 iSCSI 的恢复*：对于 iSCSI 数据存储或 LUN，SMP 从快照恢复 LUN，验证恢复的设备，跨 Proxmox 节点执行 iSCSI 重新扫描，并确保设备可访问性。对于驻留在 iSCSI 存储上的 VM，SMP 克隆快照 LUN，将克隆的 LUN 映射到 Proxmox 主机，并执行 iSCSI 发现以导入它们。然后添加恢复的存储，并更新 VM 配置。VM 磁盘被移回其原始存储，每个 VM 都跟踪进度。该过程以清理活动结束，包括移除临时存储和删除克隆的 LUN。

=== 克隆工作流详细信息

SMP 通过将繁重的工作卸载到 ONTAP 来执行存储端克隆而不是主机端文件复制，从而实现快速 VM 克隆的自动化。在底层，SMP 利用 ONTAP FlexClone 语义（基于快照、可写、节省空间的克隆），因此克隆可以快速创建，最初消耗的容量很少；它尽可能避免通过 Proxmox 主机进行数据读取/写入。

这种方法通过将克隆工作推入 ONTAP，提供适合开发/测试、实验室和快速刷新周期的快速、精简、存储效率高的克隆，从而加速 VM 配置。

默认情况下，对活动文件系统执行克隆。克隆工作流还允许选择快照，在这种情况下，使用从所选快照创建的新卷执行文件克隆。

== 摘要

Storage Manager for Proxmox 提供了一种简化和统一的方式来管理 Proxmox 环境中的保护、克隆和恢复操作。通过与 ONTAP 直接集成，该解决方案将企业级存储的效率、可靠性和性能融入到日常 VM 工作流程中。ONTAP 的节省空间的快照、快速克隆功能和内置复制提供了一致和可预测的数据保护，而 SMP 将这些功能简化为直观的、以任务为中心的界面。
